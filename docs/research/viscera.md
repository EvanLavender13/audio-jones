# Viscera

Organic, fleshy terrain generated by iterated domain warping (warp FBM). Each noise octave feeds back into the next through sinusoidal distortion, creating folded, intestinal structures. Bump-mapped Phong lighting gives the pattern a 3D surface quality - ridges cast shadows, valleys recede. Radial pulsation waves expand and contract from the center, making the surface breathe. Colored via gradient LUT with FFT brightness modulation.

## Reference Code

Buffer A - height map generator and bump-mapped lighting:

```glsl
mat2 rotate(float a) {
    float c = cos(a), s = sin(a);
    return mat2(c, -s, s, c);
}

float map(vec2 u, float t) {
    vec2 n = vec2(0.0);
    vec2 q = vec2(0.0);
    float d = dot(u, u);
    float s = 9.0;
    float o = 0.0;
    float j = 0.0;
    mat2 m = rotate(5.0);

    for (int i = 0; i < 16; i++) {
        j += 1.0;
        u = m * u;
        n = m * n;
        q = u * s + t * 4.0 + sin(t * 4.0 - d * 6.0) * 0.8 + j + n;
        o += dot(cos(q) / s, vec2(2.0));
        n -= sin(q);
        s *= 1.2;
    }

    return o;
}

void mainImage( out vec4 fragColor, in vec2 fragCoord )
{
    vec2 uv = (fragCoord - 0.5 * iResolution.xy) / iResolution.y;
    float t = iTime;

    float h = map(uv, t);

    vec2 e = vec2(2.0 / iResolution.y, 0.0);
    float hx = map(uv + e.xy, t);
    float hy = map(uv + e.yx, t);

    vec3 normal = normalize(vec3(h - hx, h - hy, 0.05));

    vec3 lightPos = normalize(vec3(0.5, 0.5, 1.0));
    vec3 viewPos = vec3(0.0, 0.0, 1.0);

    float diff = max(dot(normal, lightPos), 0.0);

    vec3 reflectDir = reflect(-lightPos, normal);
    float spec = pow(max(dot(viewPos, reflectDir), 0.0), 32.0);

    float val = max(0.0, h * 1.2);
    vec3 redish = vec3(val * 1.6, val * val * 0.9, val * val * val * 0.3);

    vec3 col = redish * (0.5 + 0.5 * diff) + vec3(1.0, 0.8, 0.6) * spec * 0.8;

    fragColor = vec4(col, 1.0);
}
```

Image pass (radial blur - NOT used in our adaptation):

```glsl
void mainImage( out vec4 fragColor, in vec2 fragCoord )
{
    vec2 uv = fragCoord / iResolution.xy;

    const int SAMPLES = 64;
    float dist = length(uv - 0.5);
    float STRENGTH = 0.02 * dist;

    vec3 acc = vec3(0.0);
    vec2 center = vec2(0.5);

    for(int i = 0; i < SAMPLES; i++) {
        float scale = 1.0 - float(i) * STRENGTH*0.05;
        vec2 scaledUV = (uv - center) * scale + center;
        acc += texture(iChannel0, scaledUV).rgb;
    }

    vec3 col = acc / float(SAMPLES);
    col *= 1.2;
    col = tanh(col);

    fragColor = vec4(col, 1.0);
}
```

## Algorithm

### Keep verbatim
- `map()` function structure: iterated domain warping with rotation matrix, sinusoidal feedback (`n -= sin(q)`), cosine accumulation (`dot(cos(q)/s, vec2(2.0))`)
- Bump mapping via finite-difference normals (3 samples of `map()`)
- Phong lighting: diffuse + specular with fixed light direction

### Replace
- `iResolution` -> `resolution` uniform
- `iTime` -> `time` uniform (phase-accumulated on CPU via `animSpeed`)
- Hardcoded constants -> config struct uniforms (see Parameters table)
- Hardcoded flesh coloring (`redish`) -> gradient LUT sampling: `t = clamp(h * heightScale, 0.0, 1.0)`, then `texture(gradientLUT, vec2(t, 0.5)).rgb`
- Add FFT brightness modulation using the standard pattern: same `t` that indexes the gradient LUT also computes FFT frequency via `baseFreq * pow(maxFreq / baseFreq, t)`
- Final color: `col = lutColor * brightness * (0.5 + 0.5 * diff) + specColor * spec * specIntensity`
- `specColor` sampled from gradient LUT at `t = 1.0` (highlight picks up the bright end of the palette)
- Skip Image pass (radial blur) entirely
- `0.05` in normal z-component -> `bumpDepth` uniform (higher = flatter surface, lower = more pronounced bumps)

### Parameter mapping
| Reference | Config field | Notes |
|-----------|-------------|-------|
| `s = 9.0` (initial scale) | `scale` | Pattern zoom level |
| `rotate(5.0)` | `twistAngle` | Rotation per iteration (radians) |
| `16` (loop count) | `iterations` | Detail level / octave count |
| `s *= 1.2` | `freqGrowth` | Frequency multiplier per octave |
| `n -= sin(q)` weight | `warpIntensity` | Scale the `sin(q)` feedback: `n -= sin(q) * warpIntensity` |
| `t * 4.0` | `animSpeed` | Phase accumulation speed (CPU-side) |
| `sin(t*4.0 - d*6.0) * 0.8` | `pulseFreq`, `pulseAmplitude` | Radial pulsation: `sin(phase * pulseFreq - d * pulseRadial) * pulseAmplitude` |
| `d * 6.0` | `pulseRadial` | Radial frequency of pulsation rings |
| `0.05` | `bumpDepth` | Normal z-component (surface flatness) |
| `32.0` | `specPower` | Specular shininess exponent |
| `0.8` | `specIntensity` | Specular highlight brightness |
| `h * 1.2` | `heightScale` | Height-to-color mapping scale |

## Modulation Candidates

- **scale**: zooms the organic pattern in/out, shifting between macro blobs and micro detail
- **warpIntensity**: low = smooth noise, high = deep folds and convolutions
- **pulseAmplitude**: controls how strongly the surface breathes
- **pulseRadial**: shifts ring density, changing the breathing texture
- **specPower**: shifts between broad matte glow and tight wet highlights
- **heightScale**: expands or compresses the color mapping across the surface

### Interaction Patterns

- **warpIntensity x pulseAmplitude (competing forces)**: Warp creates static structural folds while pulsation pushes radial waves through them. High warp + high pulse = chaotic churning where neither dominates. Low warp + low pulse = gentle undulating blobs. Audio driving these in opposition creates a tension axis from serene to violent.
- **scale x iterations (cascading threshold)**: At low scale, few iterations look smooth. Adding iterations at low scale has diminishing returns. But at high scale, each additional iteration reveals a new layer of fine detail - iterations only "pay off" visually past a scale threshold. Modulating iterations with treble energy reveals micro-texture only when high frequencies are present.

## Notes

- **Performance**: `map()` is called 3 times per pixel (center + 2 bump offsets), each doing `iterations` loop steps. At 16 iterations, that is 48 trig evaluations per pixel. Consider half-res flag if needed.
- **Phase accumulation**: `animSpeed` controls CPU-side `phase += animSpeed * deltaTime`. The shader receives `phase` as `time` uniform. `pulseFreq` multiplies inside the shader: `sin(time * pulseFreq - d * pulseRadial)`.
- **Height range**: The accumulated `o` from `map()` is unbounded but typically falls in [-3, 3]. `heightScale` normalizes this into [0, 1] for gradient LUT sampling. Values outside get clamped.
